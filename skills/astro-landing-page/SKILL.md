# Astro Landing Page & Corporate Website

Skill for building fast, SEO-optimized landing pages and corporate websites using Astro.

## Stack & Versions

- **Framework**: Astro 6.x
- **Language**: TypeScript
- **Styling**: Vanilla CSS (custom properties / CSS variables) — NO TailwindCSS unless user explicitly requests it
- **Design Source**: Figma (via MCP integration)
- **Node**: >= 22.12.0

## Project Structure

```
src/
  layouts/
    Layout.astro        # Base HTML layout with <head>, fonts, meta
  pages/
    index.astro         # Homepage
    [page].astro        # Additional pages
  components/
    Header.astro        # Site header/navigation
    Footer.astro        # Site footer
    Hero.astro          # Hero sections
    [Section].astro     # Content sections
  styles/
    global.css          # CSS variables, resets, typography
    [component].css     # Component-specific styles (optional)
  content/              # Content collections (if using CMS-like features)
public/
  images/               # Optimized images
  fonts/                # Self-hosted fonts
```

## Design Workflow (Figma → Code)

1. **Get Figma data**: Use `mcp_figma_get_figma_data` to read the design structure
2. **Download assets**: Use `mcp_figma_download_figma_images` to export images/icons
3. **Extract design tokens**: Colors, spacing, typography from Figma → CSS variables
4. **Build components**: One Astro component per Figma frame/section
5. **Pixel-perfect**: Match the Figma design as closely as possible

## CSS Architecture

### Design Tokens (global.css)
```css
:root {
  /* Colors */
  --color-primary: #...;
  --color-secondary: #...;
  --color-text: #...;
  --color-text-muted: #...;
  --color-background: #...;
  --color-surface: #...;

  /* Typography */
  --font-heading: 'Font Name', serif;
  --font-body: 'Font Name', sans-serif;
  --font-size-sm: clamp(0.875rem, 0.8rem + 0.2vw, 1rem);
  --font-size-base: clamp(1rem, 0.9rem + 0.3vw, 1.125rem);
  --font-size-lg: clamp(1.25rem, 1.1rem + 0.5vw, 1.5rem);
  --font-size-xl: clamp(1.75rem, 1.4rem + 1vw, 2.5rem);
  --font-size-2xl: clamp(2.25rem, 1.8rem + 1.5vw, 3.5rem);

  /* Spacing */
  --space-xs: 0.5rem;
  --space-sm: 1rem;
  --space-md: 1.5rem;
  --space-lg: 2rem;
  --space-xl: 3rem;
  --space-2xl: 5rem;

  /* Layout */
  --max-width: 1200px;
  --header-height: 80px;
}
```

### Component Styles
- Use scoped `<style>` blocks in Astro components
- BEM-like naming: `.hero`, `.hero__title`, `.hero__subtitle`
- Fluid typography with `clamp()`
- Responsive via container queries or media queries

## SEO Checklist

Every page MUST include:
- [ ] Unique `<title>` tag
- [ ] `<meta name="description">` 
- [ ] `<meta name="viewport">`
- [ ] Open Graph tags (og:title, og:description, og:image)
- [ ] Canonical URL
- [ ] Proper heading hierarchy (single h1, then h2, h3, etc.)
- [ ] Semantic HTML (header, main, nav, section, article, footer)
- [ ] Alt text on all images
- [ ] Schema.org structured data (LocalBusiness, Person, etc. as appropriate)
- [ ] Sitemap (auto-generated by Astro)
- [ ] Language attribute on `<html>`

## Performance Rules

1. Use Astro's built-in image optimization (`astro:assets`)
2. Self-host fonts (no Google Fonts CDN) — preload critical fonts
3. Minimize JavaScript — prefer zero-JS Astro components
4. Use `loading="lazy"` for below-fold images
5. Critical CSS should be inlined in the layout
6. Target Lighthouse score: 95+ on all metrics

## Content Management

For sites needing basic content editing:
- Use Astro Content Collections with Markdown/MDX
- Create a simple admin UI if needed (can be a separate protected route)
- Keep content in `src/content/` with proper schemas

## Deployment

- Build: `astro build`
- Output: Static files (default) or SSR if needed
- Host: Firebase Hosting, Netlify, or Vercel

## Aesthetic Standards

- Designs must feel premium and professional
- Use smooth transitions (300ms ease) on hover states
- Subtle parallax or scroll-triggered animations where appropriate
- High-quality imagery — never use placeholder images
- Consistent spacing rhythm throughout the page
- Color palette must be harmonious — extract from Figma design

## Don'ts

- ❌ Don't use React/Vue islands unless truly needed for interactivity
- ❌ Don't use TailwindCSS unless the user explicitly asks for it
- ❌ Don't load heavy JS frameworks for simple animations — use CSS
- ❌ Don't use generic stock-looking design — match the Figma faithfully
- ❌ Don't skip accessibility (ARIA labels, keyboard nav, contrast ratios)
